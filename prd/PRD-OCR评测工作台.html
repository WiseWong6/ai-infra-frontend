功能简述

为算法工程师和测试人员提供高效的 OCR（光学字符识别）模型效果评测环境，允许上传文档同时调度多款模型出结果，并在三栏视图下便捷查看“源文档”、“原始结构化数据”与“渲染还原样式”，提升模型对比与评测体验。

项目背景

当前痛点：在对不同 OCR 大模型或小模型进行效果验收与性能摸底时，存在多次上传、肉眼难溯源对比、长文本滚动难以对齐、不支持页面级精确配置等问题。

解决方案：统一的 OCR 评测工作台。集成拖拽多文件上传、配置弹窗（可选取多个模型并行运行）；在结果呈现上，利用可拖拽调整宽度的三栏视图（PDF原文、Model Output、富文本渲染），并且高度同步滚动与高亮选词情况。

功能一：文件上传与模型配置F1
P0
需求概述

支持拖出、点击上传图片及PDF。解析发起前，呼出集中配置弹窗，让用户选择要运行的 OCR 模型（可多选）并针对各类文件指定需要解析的页码范围。

功能二：侧边栏任务管理F2
P1
需求概述

一旦发起解析任务，按照「文件 × 选中模型」的维度拆分任务实例，并展示在左侧列表，以进度条和状态指示当前任务池的处理情况。

详情与交互
元素 呈现规则/交互
任务卡片 展示文件名称（截断支持）、文件大小、使用的模型名称及 Tag

卡片点击 激活该任务，右侧工作流转为该模型针对该文件的处理结果（自动加载PDF呈现对应结果）
全局进度 位于侧边栏底部。分子为完成数量，分母为总任务实例量，配合主颜色进度条
功能三：三栏对比评测视图F3
P0
需求概述

右侧主面板划分为 3 个平行窗口：PDF原始文件预览、Raw大模型原始输出数据（支持 JSON/Markdown 等）、渲染后所见即所得的图文/表格网页视图。

各栏具体说明
视窗名称 主要控件 作用与表现
面板 1：文件预览 缩放控制器、翻页器 解析PDF（使用 pdf.js）到 Canvas 渲染，底部隐藏文字层（支持高亮选词）。支持页面级缩放控制：放大、缩小、适应宽度自动缩放。
面板 2：原始数据 (Raw) 纯文本区域 代码字体的 <pre> 容器，保持换行。呈现大模型真实回包数据。 数据结构跟随模型而异：如 DeepSeek/GLM 输出结构化 Markdown，Paddle 输出带坐标点的 JSON 或嵌套 List。面板底部提供“数据格式”标识符。
面板 3：还原样式 (Render) 混合渲染区 依据面板 2 的输出进行自适应图文重绘。如果 Raw 为 Markdown，则直接解析器呈现出表格、标题等富文本结构；如果输出为 JSON (带Bbox)，则在此结合前端逻辑在空白画板上重绘版面；或者如果是混合结构，将两者进行糅合，从而实现直观的“所见即所得”对齐比测。
功能四：视图联动与响应式交互F4
P0
需求概述

极致丝滑的用户体验是平台一大核心竞争力。三栏之间具有强关联性：调整列宽与滚动阅读都能够产生联动体验。

联动特性明细

联动机制 具体交互描述
拖拽控制宽幅 栏与栏之间的空白条，悬停或拖拽时可拉伸/压缩相应视窗大小，自动重新自适应其它两栏尺寸。当拖拽释放，PDF如果选择为「自动适配宽度」将重新 Render 清晰度。
翻页同步 无论通过顶部翻页按钮、键盘左右键，还是单面板过度滚动（Overscroll 阻尼特效），只要切换页码，三个面板将同步重置并滚动到页的首部对其展示。
同页可见域感知 三个面板中任一面板处于被阅读并滚动的状态时，系统会计算并自动将顶部页码指示器更新到其占最多视区的该对应页码。
选中高亮同步 （重要！）在任一面板触发文本双击或滑动选中。只要字符数 >= 2 即可成为“查询靶标”。系统自动将该靶标在自身 PDF 文字层、Raw 文本甚至 渲染 HTML 中用 <mark> 黄色高亮标签全部标出位置，方便交叉校验错别字或是遗漏内容。释放后高亮保持，再次点击空白消失。
