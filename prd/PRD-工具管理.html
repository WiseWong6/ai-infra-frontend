功能简述

为 AI Infra 平台新增工具管理模块，让大模型节点能够调用内部 AO/HTTP 接口。通过结构化的工具定义（Schema），大模型可以理解何时需要调用工具以及如何提取参数。

项目背景

核心问题：大模型无法访问企业内部数据和 API，导致回答缺乏实时性或准确性。

解决方案

通过工具（Tools）机制，让大模型能够：

  • 调用内部 AO/HTTP API 获取真实数据
  • 执行特定操作（如发送通知、创建记录）
  • 跨系统联动，扩展能力边界
目标用户
  • AI 应用开发者：需要为大模型扩展能力的开发者
  • 运维人员：需要排查工具调用问题的运维人员

F1: 工具管理列表
P0

展示所有已注册的工具列表，支持按名称搜索、按状态筛选。包含工具的基本信息、API 配置和操作入口。

原型演示
字段说明
字段 类型 说明
ID Number 工具唯一标识
工具名称 String 英文小写字母+下划线，如 get_user_info
描述 String 给大模型看的工具用途说明
API 地址 String HTTP/HTTPS 接口地址
方法 Enum GET 或 POST
状态 Enum 启用 / 停用
创建时间 DateTime 记录创建时间
F2: 工具新建/编辑
P0

通过右侧 Drawer 抽屉表单创建或编辑工具。表单分为三个区块：基本信息、参数 Schema、API 配置。

表单区块
区块 字段 必填 说明
基本信息 工具名称 是 正则 /^[a-z_][a-z0-9_]*$/
工具描述 是 告诉大模型何时调用此工具
参数 Schema JSON Schema 是 定义工具参数的 JSON Schema
API 配置 API 地址 是 HTTP/HTTPS URL
HTTP 方法 是 GET / POST 单选
认证方式 否 无 / API Key / Bearer Token
超时时间 否 默认 30 秒
F3: 工具测试
P0

在工具编辑 Drawer 内切换到「测试」Tab，输入测试参数并执行，查看调用结果。支持查看状态码、耗时和响应体。

测试流程
  1. 在工具列表点击「测试」按钮
  2. Drawer 切换到测试 Tab
  3. 在参数输入框填写测试参数（JSON 格式）
  4. 点击「执行测试」按钮
  5. 查看返回的状态码、耗时和响应体

说明：测试调用不会记录到调用日志中，仅用于验证工具配置是否正确。

F4: 调用日志
P1

记录每次工具调用的完整请求和响应信息，支持按工具名称、调用状态、时间范围筛选。

原型演示
筛选条件
  • 工具名称：按工具名称搜索
  • 调用状态：成功 / 失败 / 超时
  • 开始时间：筛选起始时间
  • 结束时间：筛选结束时间
详情查看

点击「查看请求」或「查看响应」打开详情 Drawer（宽 80%），显示：

  • 请求详情：参数 JSON + 请求头
  • 响应详情：状态码 + 响应体 + 错误信息（若有）
数据模型
Tool 工具
字段 类型 必填 说明
id Number 是 工具唯一 ID
name String 是 工具名称，英文小写+下划线
description String 是 工具描述，给大模型看
schema JSON 是 参数 JSON Schema
apiUrl String 是 API 接口地址
method Enum 是 GET / POST
auth Enum 否 none / api_key / bearer
timeout Number 否 超时时间（秒），默认 30
status Enum 否 enabled / disabled，默认 enabled
createdAt DateTime 是 创建时间
CallLog 调用日志
字段 类型 说明
id String 调用唯一 ID
toolId Number 关联的工具 ID
toolName String 工具名称
status Enum success / error / timeout
duration Number 耗时（毫秒）
request JSON 请求参数和请求头
response JSON 响应状态码、响应体、错误信息
calledAt DateTime 调用时间
