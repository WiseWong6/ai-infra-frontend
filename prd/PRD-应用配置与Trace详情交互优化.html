Chapter 01

需求背景与目的

为什么要做这次优化

原版痛点（两部分）

1) 应用配置页中“提示词编辑”可视空间不足，配置阅读与校对效率偏低；
2) Trace追踪“查看详情”浮层中过程日志为长文本堆叠，缺少结构分层与定位能力，阅读和排障成本高。

本次优化的核心目标：

第一部分：提示词配置可视与操作效率优化

为提示词配置预留更充足空间，左侧参数与右侧提示词编辑区分工清晰，降低编辑负担。

第二部分：Trace过程日志结构化可读

将过程日志从整段文本升级为节点化阅读与快速定位能力

体验一致性与排障效率提升

统一入口、统一阅读方式，缩短定位异常路径的时间

Chapter 02

需求范围界定

澄清哪些系统界面包含在本次重构中

涉及重构的功能场景（两部分）：

  • 第一部分： 应用管理 → 应用配置（侧推抽屉版） → 点击表格行的“提示词编辑”（文件：frontend-app-config.html）
  • 第一部分： 应用管理 → 应用配置（独立视效跳转页） → 点击表格行的“提示词编辑”（文件：frontend-app-config-edit.html）
  • 第二部分： AI Infra → Trace追踪列表 → 点击“查看详情” → 过程日志Tab（文件：frontend-trace.html）

不包含在本次迭代的功能：

后端Trace采集链路、日志原始字段协议、历史数据回灌与离线修复能力不在本次交互改版范围内。

优化背景

原有 Trace 列表页仅提供单一"查看详情"入口，无法快速直达不同信息维度。本次优化将操作列升级为"三按钮组"模式，同时增加单元格点击快捷跳转能力，提升信息获取效率。

优化目标

快速直达

从列表直接定位到详情页的指定 Tab，减少二次点击。

单元格跳转

点击特定单元格直接打开对应信息，符合直觉操作。

交互原型预览 打开Trace列表原型

交互规则说明

操作列：三按钮组

将原有的单一"查看详情"链接升级为三个独立按钮，每个按钮直接打开详情浮层并定位到对应 Tab：

  • 运行结果：打开详情浮层 → 自动切换到"运行结果"Tab
  • 过程日志：打开详情浮层 → 自动切换到"过程日志"Tab
  • 反馈质检：打开详情浮层 → 自动切换到"反馈质检"Tab
单元格点击跳转规则

不同类型的单元格支持点击快捷跳转，具体规则如下：

  • 质检单元格（评价、备注、标签）：点击后打开详情浮层 → 自动切换到"反馈质检"Tab
  • 输入/输出单元格：点击后打开详情浮层 → 自动切换到"运行结果"Tab
  • ID/项目/应用单元格：点击后打开详情浮层 → 默认展示"运行结果"Tab（首 Tab）

验收标准

  • 操作列三个按钮均可独立点击并正确定位到对应 Tab
  • 单元格点击跳转符合上述规则定义
  • 按钮与单元格点击不影响行选中等其他交互
当前实现痛点

当前"过程日志"为超长文本堆叠，信息密度高且缺少结构分层；用户在查看一次追踪时，难以快速定位关键步骤、失败点与上下文关系，阅读成本高，排障效率低。

版本拆分说明

A版（当前样式）：保留 Timeline + 左侧目录导航结构；B版（简化样式）：取消过程日志目录，日志以大文本块展示，默认超过 3 行收起，点击展开查看完整内容。

优化目标

结构化阅读

将过程日志从"长文本"改为"分段节点"，按阶段展示，支持快速扫读。

快速定位问题

提供搜索、时间点、异常高亮与节点跳转，降低排障定位时间。

稳定兼容现有流程

仅优化展示与交互层，不改Trace采集链路与原始日志字段定义。

3.3-A 详细规则说明（当前样式：Timeline + 目录）

1. 浮层信息架构

详情浮层采用"双栏布局 + 条件侧滑"设计，从左到右依次为：

  • 左侧节点目录（180px 固定宽度）：固定侧边栏，sticky 定位，滚动时始终可见
  • 中央 Timeline 主阅读区（自适应宽度）：卡片化展示过程日志，包含时间轴、节点卡片、引用卡片等

头部固定区：TraceID、运行状态、请求标识（userId/groupId），支持一键复制。
条件侧滑：点击引用卡片时，从右侧滑出知识库切片详情面板（详见 3.4 章节）。

2. 左侧节点目录（Timeline Nav）

固定侧边栏目录，提供全局导航与状态预览能力：

  • 节点列表：按时间顺序展示所有节点标题
  • 点击跳转：点击任意节点标题，主阅读区自动滚动至对应卡片顶部
  • 滚动高亮同步：当用户滚动主阅读区时，目录中对应的节点标题自动高亮（蓝色背景），实现双向联动
3. Timeline 视图（过程日志主阅读区）

将日志从"长文本堆叠"升级为"时间轴卡片化视图"，每个节点卡片包含：

  • 时间戳 + 阶段标签：如 "10:55:55.123 · Prompt构建"
  • 节点卡片：包含标题、元数据（Tokens/耗时）、内容区
  • 引用卡片组：当节点包含知识库引用时，展示紧凑型引用卡片列表（点击触发 3.4 章节的侧滑面板）

长文本默认折叠（约 20 行），点击"展开全部内容"查看完整信息。

4. 内容格式化（自动识别与渲染）

系统自动识别节点内容的格式类型，并采用对应的渲染方式：

  • JSON 格式化：自动检测 JSON 格式内容，进行语法高亮 + 折叠展开 + 行号显示
  • Markdown 渲染：自动检测 Markdown 格式（如含 # 标题、**加粗**、- 列表等），渲染为富文本展示
  • HTML 渲染：自动检测 HTML 标签，进行安全渲染（过滤 script 等危险标签）
  • 纯文本兜底：无法识别格式时，展示原文

切换能力：提供"原始/渲染"切换按钮，用户可随时查看原始文本。

5. 复制能力

提供多层级复制能力，便于排障沟通：

  • 复制当前节点：复制单个节点卡片的内容（含标题、时间戳、原始文本）
  • 复制全部过程日志：复制所有节点的完整内容
场景 处理方式
加载中 显示骨架屏，不阻塞关闭与 Tab 切换
无日志数据 显示空态文案与原因提示
拉取失败 显示错误态，支持重试按钮
性能要求 首屏加载 ≤ 1.5s；大体量日志保持滚动流畅

验收标准

  • 用户可在 10 秒内定位到失败步骤或异常节点。
  • 长日志默认不再整段铺开，首屏展示信息层级清晰。
  • 左侧目录：点击可精确滚动至对应卡片；滚动时目录高亮同步更新；
  • JSON/Markdown/HTML 内容自动识别并格式化渲染，支持切换查看原始文本。
  • 复制节点与复制全部内容可直接用于排障沟通，不丢关键上下文。

3.3-B 详细规则说明（简化样式：无目录日志块）

1. 布局与展示
  • 无目录节点：取消左侧过程日志目录，仅保留主阅读区。
  • 日志样式：日志内容以大文本块展示（贴近当前原始日志观感，如示意图）。
  • 保留头部信息：TraceID、运行状态、请求标识与复制能力保持不变。
2. 三行收起与展开
  • 默认策略：单条日志默认展示 3 行。
  • 交互行为：超过 3 行时显示“展开”，展开后可“收起”。
  • 短内容：不足 3 行不展示展开按钮。
3. 格式解析（沿用优化能力）
  • 保留 JSON/Markdown/HTML 自动识别与渲染能力。
  • 无法识别时按纯文本展示。
  • 格式解析与 3.5 章节保持一致口径。

3.3-B 验收标准

  • 过程日志区域不出现左侧目录节点。
  • 单条日志超过 3 行时默认收起，并支持展开/收起切换。
  • 日志大文本块样式下，JSON/Markdown/HTML 仍能正确解析与展示。
  • 切片引用入口可正常触发 3.4 面板。
功能定位

当 Trace 过程日志中的节点包含知识库引用时，用户可点击引用卡片查看切片详情。本章节定义切片查看的交互方式与浮层逻辑。

版本适配说明

3.4-A 适配 3.3-A（Timeline + 目录）；3.4-B 适配 3.3-B（无目录日志块）。两版均使用同一套切片侧滑面板样式与交互能力。

交互原型预览（3.4-B：配3.3B） 打开3.4-B原型

交互规则说明

触发入口

两个版本的触发入口分别为：

  • 3.3-A：在 Timeline 视图中，节点卡片内的引用卡片组触发。
  • 3.3-B：在无目录日志块中，文本块下方的引用入口触发。
  • 引用卡片：紧凑型卡片，展示切片名称 + 图标
  • 点击行为：点击任意引用卡片，触发二级侧滑面板从右侧滑出
  • 多引用场景：当一次引用多个切片时，卡片组以网格形式排列，点击任意一个均可打开面板
二级侧滑面板（KB Slice Panel）

从右侧滑出的知识库切片详情面板，与 3.3 浮层的关系：

  • 面板宽度：800px，从屏幕右侧滑入
  • 层级关系：覆盖在 3.3 详情浮层之上，但不关闭浮层
  • 面板头部：
    • 展示切片标题
    • 全局复制按钮（下拉菜单："复制当前切片" / "复制全部切片"）
    • 关闭按钮（X）
  • 内嵌导航：当一次引用多个切片时，面板内左侧提供切片导航目录（A/B 两版均一致）
面板内容区

展示知识库切片的完整内容：

  • 切片内容：支持 Markdown 渲染（与 3.3 格式化规则一致）
  • 元信息：展示切片来源（知识库名称）、切片内容等
  • 来源链接：提供"查看源文件"按钮，跳转至知识库管理页对应源文件并选中对应切片
关闭方式
  • 点击面板头部关闭按钮（X）
  • 点击 3.3 详情浮层区域（面板外的任何位置）
  • 按 Esc 键（优先关闭面板，再次按 Esc 关闭浮层）

验收标准

  • 3.3-A/3.3-B 两个版本中的引用入口均可正确触发二级侧滑面板。
  • 面板宽度 800px，层级正确（覆盖但不关闭 3.3 浮层）。
  • 多切片场景下，面板内导航目录可正常切换切片内容。
  • 三种关闭方式均可正常工作。
  • 复制功能支持"复制当前切片"和"复制全部切片"。
功能定位

在 Trace 追踪的"运行结果"Tab 中，为输入（Input）和输出（Output）内容增加 JSON 格式自动解析与渲染功能，与 3.3 章节过程日志的格式化能力保持一致。

交互原型预览 打开运行结果原型

格式化规则说明

自动格式检测

系统自动检测输入/输出内容格式：

  • JSON 格式：以 { 或 [ 开头且可被 JSON.parse 解析的内容
  • Markdown 格式：包含 #、**、-、``` 等 Markdown 标记
  • HTML 格式：包含 < 和 > 标签对
  • 纯文本：以上均不匹配时，按原始文本展示
渲染效果

不同格式内容的渲染方式（与 3.3 章节保持一致）：

  • JSON 格式：
    • 折叠展开功能（默认收起过长内容，点击"展开内容"展开）
    • 缩进格式化（2 空格）
  • Markdown 格式：
    • 标题渲染（# → H1，## → H2）
    • 加粗、斜体渲染
    • 列表渲染（无序）
    • 代码块渲染（行内 ` 和块级 ```）
  • HTML 格式：安全渲染（转义 HTML 标签，防止 XSS）
  • 纯文本：保留换行和空格格式
高度限制

输入/输出区域的最大高度限制：

  • 最大高度：约 30 行（约 240px）
  • 超出处理：默认收起并显示“展开内容”，展开后内容区域内部出现垂直滚动条
  • 与过程日志一致：高度限制规则与 3.3 章节过程日志保持一致
格式切换控件

在输入/输出区域头部提供格式切换下拉框：

  • 选项：
    • 混合格式：根据内容自动检测并渲染（JSON/Markdown/HTML/纯文本）
    • HTML格式：强制 HTML 渲染
    • JSON格式：强制 JSON 格式化渲染
    • MD格式：强制 Markdown 渲染
  • 默认选项：混合格式
  • 位置：输入区域和输出区域各自独立的下拉框

验收标准

  • 输入/输出内容自动识别 JSON 格式并进行格式化渲染（折叠展开）。
  • 非 JSON 内容保持原始展示（或按 Markdown/HTML 渲染）。
  • 提供格式切换能力（自动/HTML/JSON/Markdown），切换后立即生效。
  • 首屏加载性能 ≤ 1.5s。
  • 渲染效果与 3.3 章节过程日志的格式化保持一致。

交互原型

可交互的界面原型预览与快速访问
