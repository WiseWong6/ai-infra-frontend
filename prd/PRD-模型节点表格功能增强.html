项目背景

现状问题：

  • 模型节点表格无法直观查看各节点的提示词内容，需进入编辑页面才能查看
  • 切换模型需要先选中节点，再点击批量切换按钮，操作路径较长
  • "资源节点"命名不够准确，实际为资源配置
  • 操作按钮布局不够紧凑，影响整体表格可读性
目标
  • 提升提示词查看效率，减少页面跳转
  • 简化模型切换操作，支持单节点快速切换
  • 统一命名规范，提升系统可理解性
  • 优化UI布局，提升整体视觉体验

F1: 提示词列展示
FEAT-001
P1
功能概述

在模型节点表格中新增"系统提示词"和"用户提示词"两列，展示各节点的提示词内容预览，支持鼠标悬停查看完整内容。

需求明细
字段 说明
系统提示词 展示 resourcePromptSnapshot.messages 中 role 为 system 的消息内容，截取前50字符，超出显示"..."
用户提示词 展示 resourcePromptSnapshot.messages 中 role 为 user 的消息内容，截取前50字符，超出显示"..."
列宽 固定 150px
悬停提示 鼠标悬停时通过 title 属性显示完整提示词内容
空值处理 无提示词时显示"-"
表格列调整
列名 宽度 说明
复选框 40px 全选/单选
ID 80px 资源ID
名称 自适应 资源名称
系统提示词 150px 新增 - 提示词预览
用户提示词 150px 新增 - 提示词预览
当前模型 140px 支持点击切换
最近更新角色 自适应 更新人
最近更新时间 140px 更新时间
操作 100px 编辑/删除
F2: 模型内联切换
FEAT-002
P1
功能概述

将"当前模型"列从纯文本展示改为可点击的下拉选择器，支持直接在表格行内切换模型，无需进入编辑页面。

交互逻辑
交互 行为
点击模型名称 展开下拉选择框，显示可用模型列表
再次点击模型名称 收起下拉选择框（toggle 行为）
点击下拉框外部 收起下拉选择框
选择新模型 立即保存变更，显示 Toast 提示，刷新表格数据
选择相同模型 仅收起下拉框，不触发保存
视觉样式
元素 样式
模型名称 蓝色文字 (#3b82f6)，浅蓝色背景 (rgba(59,130,246,0.08))，圆角 4px
悬停效果 背景色加深 (rgba(59,130,246,0.15))
下拉框背景 白色 (#fff)
下拉框边框 灰色边框，8px 圆角，阴影效果
数据流
读取
从 localStorage[app_resource_bindings_demo_v1] 读取 resourcePromptSnapshot.model
保存
调用 setResourceModel(resourceId, newModel, '内联切换模型') 更新数据
历史
自动记录版本历史到 app_resource_revisions_demo_v1
