Chapter 01

需求背景与目标

需求背景

1、当前 Trace 列表缺乏状态、耗时、Token、成本。耗时需要计算，状态无法选择。同时缺乏成本统计消耗价格。

2、过程日志长日志场景，展开后收起不便


解决方案

1）增加状态、耗时、Token、成本指标，区分端到端及单一模型节点。

2）过程日志增加最大展开行

涉及页面：frontend/pages/frontend-trace-prd-metrics.html

历史基线：PRD-应用配置与Trace详情交互优化.html

Chapter 02

需求详情

Trace 列表增加状态、耗时、成本指标

原型预览：Trace 列表 打开原型

1、Traceid 列：增加状态 icon，代表端到端成功或失败。成功/失败 icon 放在 TraceId 前，二者合并为一列。

2、时间列：前置，Trace 创建时间前置，成为列表最先扫读的关键信息列。

3、指标列：在输入/输出列之后增加端到端耗时、总 Token、总成本。

4、统计逻辑

1）Token 及成本，由 API 返回数据

2）端到端耗时，由末端输出完毕节点时间 - 首端节点请求时间

Trace 详情增加状态、耗时、成本指标

原型预览：详情摘要位置与样式 打开详情原型

摘要信息放在 Trace追踪 标题下方，数据与端到端 Trace 一致。

节点头部增加指标

原型预览：节点头部指标展示 打开详情原型

1）所有节点头部具备状态、耗时

2）模型节点头部额外增加总 Token、成本

3）复制按钮交互优化：复制按钮调整至日志右上角

4）Token、成本，统计逻辑由模型 API 返回

5）耗时统计：已有 A 节点，下一节点为 B，A 节点耗时 = A 节点请求时间 - B 节点请求时间

6）首个节点、末端节点没有耗时字段

展开收起交互优化

原型预览：长日志收放 打开长日志原型

长日志展开后，用户继续滚动正文时，如果收起入口跟着正文一起离开可视区，会打断阅读与排障节奏。该问题在一条 Trace 中出现多个长日志节点时会被放大。

交互规则：

1）展开默认最多展示 20 行，滑动仅在固定高度文本框内滚动，如超出该行日志区域，才触发全局滑动。

2）多个日志块彼此独立控制，不共享展开/收起状态。
